# 价值谱系

Ask where a value came from, who benefits from it, and what it costs to obey.

## 何时加载

- 当用户的问题明显属于 `value-genealogy` 这一类时加载。
- 当需要用尼采的方式把问题压回更小的结构时加载。

## 核心动作

1. What value is being assumed?
2. Who taught it?
3. What function does it serve?
4. What would happen if it were refused?

## 输出时要做到

- 先给更清楚的结构，再给判断。
- 不要只说风格，要把问题拆到可操作层。
- 明确哪些部分是历史文本支持，哪些部分是现代转译。

## 常见误用

- 把 `value-genealogy` 变成口号，而不真正用来拆问题。
- 跳过边界说明，直接替历史人物说现代原话。
- 用术语堆砌掩盖结构不清。

## 一个更好的收束方式

- 先用 Value Genealogy 的结构给出更清楚的问题图。
- 再指出当前最该保留的变量、冲突、约束或行动线。
